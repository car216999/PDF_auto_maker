2. 제안 배경 및 필요성 
1) 생성형 AI 도입 가속 
국내 749개 기업 조사 결과 55.7%가 생성형 AI를 전사(22.4%) 또는 일부 부서(33.2%)에서 활용 중이며, 
2026년에는 활용 기업이 85%를 넘어설 전망임. 활용 목적 1위는 업무 효율성·생산성 향상(70.5%)임.

2) 보안 우려의 심화 
도입 확대와 동시에 보안 우려도 커지고 있음. 국내 IT·보안 실무자 667명 조사에서 ‘AI 기반 보안 위협’이 
2026년 최대 위협 1위(81.2%)로 꼽혔으며, 생성형 AI 활용 기업의 주요 우려로 결과 신뢰도 부족(61.3%)과 
보안·개인정보 유출(53.3%)이 지목됨.

3) 온프레미스 전환 흐름 
외부 상용 API(ChatGPT 등) 사용 시 사내 기밀·견적·개인정보가 외부 서버로 전송되는 리스크가 상존함. 이에 
기업 추론을 자체 인프라에서 수행하는 비중이 2023년 12%에서 2025년 55%로 4.6배 증가함. 로컬 오픈웨
이트 모델은 토큰당 비용이 상용 API 대비 최대 18배 저렴한 것으로 보고됨.

데이터 주권을 확보하면서 정형 문서 작성 생산성을 높일 수 있음. 본 프로젝트는 이 두 목표를 동시에

기대효과 
– 업무 생산성 향상 : 정형 문서 작성 시간을 MVP 50%(중장기 70%) 단축해 핵심 업무 집중. 
– 데이터 보안 확보 : 외부 API 미사용·완전 로컬 구조로 기밀 데이터 유출 원천 차단. 
– 인프라 비용 절감 : 양자화 오픈소스 모델로 상용 API 호출 비용 대비 운영비 최소화. 
– 실용 AI 역량 내재화 : 파싱·RAG·서빙·주입까지 연동한 LLM 애플리케이션 개발 역량 확보. 
활용방안 
– 정부지원사업·공공 문서 : 규격이 까다로운 신청서·사업계획서 초안 자동 완성.
