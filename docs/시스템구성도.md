## 2. End-to-End 처리 흐름

```mermaid
flowchart LR
    A["① 빈 양식<br/>업로드"] --> B["② PDF 파싱<br/>필드·좌표 추출"]
    B --> C["③ RAG 검색<br/>사내 문서 근거"]
    C --> D["④ LLM 생성<br/>필드별 값 + 근거판정"]
    D --> E["⑤ 좌표 한글 주입<br/>평탄화"]
    E --> F["⑥ 미리보기·다운로드<br/>근거/추론 표시"]

    style A fill:#eef2fb,stroke:#1f3a5f
    style F fill:#e7f6ec,stroke:#1a8a4a
```

| 단계 | 모듈 | 담당 | 산출물 |
|---|---|---|---|
| ② 파싱 | parsing | 채요한 | FormField(이름·라벨·타입·좌표) |
| ③ 검색 | rag | 박은선 | RetrievedChunk(근거·출처) |
| ④ 생성 | generation | 이권형 | FilledField(값·grounded) |
| ⑤ 주입 | injection | 이권형·채요한 | 완성 PDF |
| ①⑥ UI·통합 | api·frontend | 김세경 | 웹 대시보드 |

## 3. 배포 구성 (docker-compose)

```mermaid
flowchart TB
    subgraph compose["docker compose up"]
        direction TB
        O["ollama<br/>(LLM·임베딩 서빙)"]
        OI["ollama-init<br/>(모델 1회 다운로드)"]
        BK["backend<br/>(FastAPI :8000)"]
        FR["frontend<br/>(nginx :5173 → /api 프록시)"]
    end
    VOL[("ollama_models 볼륨")]
    DV[("./data 마운트<br/>지식·인덱스 영속화")]

    O -.->|"healthy"| OI
    OI -.->|"완료"| BK
    BK -.->|"healthy"| FR
    O --- VOL
    BK --- DV
```

> 기동 순서: `ollama` → `ollama-init`(모델 받기) → `backend`(인덱싱·API) → `frontend`. 모델 다운로드 시점 외에는 외부 통신 없음.

## 기술 스택 요약

| 영역 | 기술 |
|---|---|
| 프론트엔드 | React + Vite, nginx(배포) |
| 백엔드 | Python 3.13, FastAPI, uv |
| PDF | PyMuPDF(파싱·주입), pypdf, ReportLab |
| RAG | Qdrant(하이브리드), BGE-M3(1024d, dense)+BM25(sparse, RRF), cross-encoder 리랭킹(jina) |
| LLM | Ollama + Qwen3 8B (Apache-2.0) |
| 평가 | 소켓 가로채기 기반 외부호출 실측 + KPI 하네스 |
