# 빅데이터 분석 정의서

주제: LLM·RAG 기반 지능형 문서처리(IDP) 에이전트 (뚝딱, TookTak)
작성일: 2026.06.22

---

## Ⅰ. 개요

### 1. 아이디어 주제
LLM과 RAG(검색증강생성) 기반 지능형 문서처리(IDP) 에이전트

### 2. 개발 목표
빈 PDF 양식과 사용자의 컨셉 입력만으로, 사내 문서를 근거로 양식의 각 항목을 자동으로 채운 완성 문서를 생성한다. 모든 처리를 사내(로컬)에서 수행해 외부 네트워크 호출 0건을 달성하고, 정형 문서 작성 시간을 단축하면서 데이터 유출 위험을 제거한다.

### 3. 개발 내용
- 사내 지식 문서를 정제·청킹·임베딩하여 벡터 데이터베이스(Qdrant)로 구축
- 질의(필드 라벨 + 컨셉)에 대한 top-k 벡터 유사도 검색(RAG)으로 근거 추출
- 온프레미스 오픈소스 LLM(Qwen3 8B)으로 근거 기반 필드값 생성 (환각 제어)
- PyMuPDF로 양식의 폼필드·좌표 추출, 생성 값을 한글 폰트로 좌표 주입·평탄화
- 생성 값의 근거/추론 판정 및 KPI(정확도·근거율·외부호출) 자동 측정

## Ⅱ. 기능별 빅데이터 분석 명세서

**기능명: 근거 기반 양식 자동 작성 (RAG 파이프라인)**

### 1. 데이터 준비

| 항목 | 내용 |
|---|---|
| 데이터 정의 | 사내 지식 문서(회사 정보·단가/규정·프로젝트 기획서 등) 및 표준 양식 PDF(견적서·사업계획서 등) |
| 데이터 획득 방법 | 사내 문서 수집 + 공공 오픈데이터셋(AI Hub 등) 및 행정·비즈니스 표준 양식 활용 (출처·라이선스 명시) |
| 수집 데이터 | 텍스트 문서(.md/.txt/PDF), AcroForm 양식 PDF |

### 2. 전처리

| 항목 | 내용 |
|---|---|
| 전처리 과정 | 1) 텍스트 정제 — 머리말·꼬리말·페이지번호 등 노이즈 제거<br/>2) 청킹 — RecursiveCharacterTextSplitter(chunk_size 800 / overlap 120, 한국어 친화 구분자)<br/>3) 빈 청크·중복 청크 필터링<br/>4) 개인정보·기밀 점검(민감정보 미포함 확인) |
| 전처리 데이터 | 정제·청킹된 텍스트 청크 집합 (출처 메타데이터 포함) |

### 3. 데이터 분석

| 항목 | 내용 |
|---|---|
| 데이터 분석 목표 | 질의(채울 필드 라벨 + 사용자 컨셉)에 가장 관련 있는 근거 청크를 사내 문서에서 검색 |
| 데이터 분석 시나리오 (알고리즘) | 1) BGE-M3(1024차원, dense) 임베딩 + BM25(sparse)<br/>2) Qdrant dense+sparse 하이브리드 검색 → RRF 융합 후보 N개<br/>3) cross-encoder(jina multilingual) 리랭킹으로 top-k 정밀 재정렬 |
| 데이터 분석 결과 | 질의별 top-k 근거 청크 + 유사도 점수 + 출처 반환 |

### 4. 모델 생성 및 학습

| 항목 | 내용 |
|---|---|
| 모델링 목표 | 검색된 근거 안에서 양식 필드 값을 생성하여 환각을 제어하고, 필드 간 계산(예: 공급가액=단가×수량)을 수행 |
| 학습 모델 | 생성: Qwen3 8B (Apache-2.0, Q4 양자화) / 임베딩: BGE-M3 — 모두 사전학습 오픈소스 모델을 로컬(Ollama) 서빙 |
| 학습 방법 | 별도 파인튜닝 없이 **RAG(검색증강생성) + 프롬프트 엔지니어링**으로 사내 문서를 근거로 주입하는 방식. (중장기: 도메인 데이터 파인튜닝·양자화 튜닝 검토) |
| 모델 예측 결과 | 필드별 생성 값 + 근거(grounded)/추론 구분 표시 + 신뢰도 |

### 5. 검증

| 항목 | 내용 |
|---|---|
| 분석 검증 방안 | 정답 라벨이 있는 양식 평가셋으로 ① 폼필드 채움 정확도 ② 근거 일치율 ③ 문서 작성 시간 단축 ④ 외부 호출 0건(소켓 트래픽 실측)을 KPI 하네스로 자동 측정. 동일 입력 반복으로 재현성 확보 |
| 분석 평가 결과 | 채움 정확도 93~100% · 근거 일치율 88~100% · 시간 단축 98%↑ · 외부 호출 0건 (3종 양식 실측) |

---

> 참고: 본 시스템은 통계·예측형 빅데이터 분석이 아니라 **텍스트 임베딩·벡터 검색(RAG) 기반 분석**으로, 사전학습 LLM을 추가 학습 없이 사내 데이터로 근거화하여 활용한다. 관련 문서: [시스템 구성도](시스템구성도.md) · [요구사항 정의서](요구사항정의서.md)
