# 월요일 — 최종기획서로 예시 교체 가이드

현재 데모/평가는 **약식기획서**(내용 있는 버전)를 지식으로 사용 중이다.
월요일에 **내용을 채운 최종기획서**가 준비되면, 아래 3단계로 교체하면 끝난다.

## 교체 절차

```bash
# 1) 내용을 채운 '최종기획서_뚝딱.pdf' 를 프로젝트 루트에 둔다 (같은 파일명).
#    위치: C:\Users\smhrd\Desktop\tooktak\최종기획서_뚝딱.pdf

cd backend

# 2) 지식 재추출 — 내용 있는 기획서를 자동 선택 (최종기획서 우선)
uv run python -m scripts.extract_plan
#    [완료] 최종기획서_뚝딱.pdf -> data/knowledge/tooktak_plan.md (...자, 키워드 N회)
#    ↳ '[경고] ... 빈 템플릿' 이 뜨면 아직 내용이 안 채워진 것 — 채운 PDF인지 확인.

# 3) KPI 재측정 + 시연 스크린샷 재생성
uv run python -m scripts.run_eval
uv run python -m scripts.make_screenshots
```

## 동작 원리

- `extract_plan` 은 **프로젝트 키워드(뚝딱·Qwen·엑시오 등)가 있는 문서를 자동 선택**한다.
  - 최종기획서가 채워져 있으면 → 최종기획서 채택
  - 비어 있으면 → 약식기획서로 폴백 (현재 상태)
- 빈 템플릿이면 경고만 하고 **기존 지식을 덮어쓰지 않는다** (안전장치).

## 교체 후 반영되는 곳

- `data/knowledge/tooktak_plan.md` (RAG 지식)
- `사업계획서` 데모·KPI (`scripts.run_eval`, `scripts.make_screenshots`)
- README 시연 스크린샷 `docs/screenshots/03_business_plan.png`

> 참고: `최종기획서_뚝딱.*` 는 `.gitignore` 로 저장소에서 제외되어 있다(로컬에만 보관).
> 추출된 텍스트(`tooktak_plan.md`)만 커밋된다.
