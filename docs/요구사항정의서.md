# 뚝딱(TookTak) 요구사항 정의서

PDF 양식 자동 작성 시스템. 빈 양식 + 컨셉 입력 → 완성 문서. **외부 호출 0건, 완전 로컬.**

> 우선순위 — **필수**(MVP 범위) · 권장 · 선택(중장기). 검증: 자동 테스트/KPI/수동 확인.

---

## 1. 기능 요구사항 (Functional)

| ID | 구분 | 요구사항 | 상세 설명 | 우선순위 | 관련 모듈 |
|---|---|---|---|:---:|---|
| RF-01 | 입력 | 양식 업로드 | AcroForm 입력 필드가 있는 빈 PDF 양식을 업로드한다 | 필수 | api |
| RF-02 | 파싱 | 폼필드 자동 추출 | 필드명·라벨·타입·좌표(bbox)·필수여부·최대길이를 추출한다 | 필수 | parsing |
| RF-03 | 파싱 | 라벨 자동 추론 | 위젯 주변 텍스트(좌측·상단)로 사람이 읽는 라벨을 추론한다 | 필수 | parsing |
| RF-04 | 입력 | 컨셉 입력 | 작성할 문서의 컨셉·지시를 자연어로 입력한다 | 필수 | api·frontend |
| RF-05 | 검색 | RAG 근거 검색 | 사내 지식 문서에서 top-k 근거 청크를 검색한다 | 필수 | rag |
| RF-06 | 생성 | 필드값 자동 생성 | 근거·컨셉 기반으로 각 필드 값을 생성한다(필드 간 계산 포함) | 필수 | generation |
| RF-07 | 생성 | 근거/추론 판정 | 생성 값이 근거(사내문서·컨셉)에 존재하는지 표시한다 | 필수 | generation |
| RF-08 | 검수 | 미리보기·값 수정 | 생성 값을 근거/추론 구분으로 보여주고 직접 수정한다 | 필수 | frontend |
| RF-09 | 주입 | 좌표 한글 주입 | 추출 좌표에 한글 폰트로 값을 주입하고 평탄화한다 | 필수 | injection |
| RF-10 | 주입 | 멀티라인 문단 채움 | 서술형 박스는 줄바꿈·폰트 자동축소로 채운다 | 필수 | injection |
| RF-11 | 출력 | 완성 PDF 다운로드 | 채워진 PDF를 내려받는다 | 필수 | api·frontend |
| RF-12 | 지식 | 지식 문서 인덱싱 | 사내 문서를 정제·청킹·임베딩해 ChromaDB에 저장한다 | 필수 | rag |
| RF-13 | 평가 | KPI 측정 | 정확도·시간단축·근거율·외부호출을 자동 측정한다 | 권장 | eval |
| RF-14 | 주입 | 평면(비-AcroForm) PDF 주입 | 입력 필드 없는 PDF에 좌표 오버레이로 주입한다 | 선택 | injection |
| RF-15 | 검색 | 하이브리드 검색·리랭킹 | BM25+벡터(RRF) + cross-encoder 리랭킹으로 정확도 향상 | 선택 | rag |

## 2. 비기능 요구사항 (Non-Functional)

| ID | 항목 | 요구사항 | 목표/기준 | 검증 |
|---|---|---|---|---|
| RN-01 | 보안 | 외부 네트워크 호출 0건 (완전 로컬) | 외부 도메인 호출 = 0 | 소켓 실측 (eval) |
| RN-02 | 보안 | 민감정보 비저장 | 학습/인덱싱 데이터에 민감정보 미포함 | 전처리 점검 |
| RN-03 | 정확성 | 폼필드 채움 정확도 | ≥ 80% (필드 일치율) | KPI 하네스 |
| RN-04 | 신뢰성 | 근거 일치율 (환각 제어) | ≥ 90% (채운 값 기준) | KPI 하네스 |
| RN-05 | 효율성 | 문서 작성 시간 단축 | ≥ 50% (수작업 대비) | KPI 하네스 |
| RN-06 | 성능 | 로컬 추론 자원 | 단일 GPU/CPU, 소형 양자화 모델(Qwen3 8B Q4 ~6GB) | 실행 환경 |
| RN-07 | 사용성 | 웹 UI 3단계 | 업로드 → 컨셉 → 미리보기·다운로드 | 수동 확인 |
| RN-08 | 호환성 | 한글 렌더링 | 한글 폰트 깨짐 없이 좌표 주입 | 출력 검증 |
| RN-09 | 이식성 | 컨테이너 배포 | docker compose 단일 명령 기동 | 배포 구성 |
| RN-10 | 유지보수성 | 모듈 인터페이스 분리 | 스키마 계약 기반 단방향(DAG) 의존 | 코드 구조 |
| RN-11 | 품질 | 자동 테스트 | 핵심 로직 오프라인 테스트 통과 | 테스트 38개 |

## 3. 제약 사항 (Constraints)

| ID | 제약 |
|---|---|
| C-01 | 외부 상용 API(ChatGPT 등) 미사용 — 사내 데이터 외부 전송 금지 |
| C-02 | 상업 배포 시 Apache-2.0 모델(Qwen3)만 사용 (EXAONE 등 비상업 라이선스 제외) |
| C-03 | MVP 지원 양식은 AcroForm 기반 (평면 PDF는 중장기) |
| C-04 | 데이터·모델 라이선스·출처 명시, 재배포 조건 준수 |

## 4. 액터별 요구사항 요약

| 액터 | 핵심 요구사항 |
|---|---|
| 사용자 (문서 작성자) | RF-01·04·08·11 — 양식 올리고 컨셉 입력해 완성 문서 받기·검수 |
| 운영자 (지식·평가 관리) | RF-12·13 — 지식 인덱싱·KPI 측정 |
