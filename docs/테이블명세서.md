# 뚝딱(TookTak) 테이블 명세서

PDF 양식 자동 작성 시스템 DB 설계. DDL 원본: [`erd.txt`](erd.txt) · 표기: PK=기본키, FK=외래키, N=NULL 허용

> 표기 약속 — PK: 기본키 / FK: 외래키 / NN: NOT NULL / U: UNIQUE

---

## 관계(Relationship) 요약

| 부모 | 자식 | 관계 | 의미 |
|---|---|:---:|---|
| users | forms | 1:N | 사용자가 여러 양식 업로드 |
| forms | form_fields | 1:N | 양식에서 여러 필드 추출 |
| forms | documents | 1:N | 한 양식으로 여러 문서 생성 |
| users | documents | 1:N | 사용자가 여러 문서 생성 |
| documents | filled_fields | 1:N | 한 문서에 여러 채운 값 |
| knowledge_documents | knowledge_chunks | 1:N | 원본 문서를 여러 청크로 분할 |
| filled_fields | knowledge_chunks | **N:M** | 채운 값 ↔ 근거 청크 (`document_evidence` 경유) |

> **핵심 설계 의도** — `document_evidence`(N:M)가 "어떤 채운 값이 어떤 사내 문서 청크에 근거했는지"를 추적해, 환각 제어·근거 검수를 데이터 모델 레벨에서 보장한다.
