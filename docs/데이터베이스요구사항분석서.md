# 데이터베이스 요구사항 분석서

과제명: 오픈소스 LLM·RAG 기반 PDF 양식 자동 작성 시스템 (뚝딱, TookTak)
작성일: 2026. 06. 22.

---

## 1. 요구사항 정의서

1) 사용자 계정 정보를 저장하고, 업로드한 양식·생성 문서의 소유자를 식별한다.
2) 업로드된 빈 양식과 파싱으로 추출한 폼필드(필드명·라벨·타입·좌표·필수여부·최대길이)를 저장한다.
3) 사내 지식 문서와 이를 청킹한 텍스트 조각을 저장하고, 벡터 검색의 대상으로 관리한다. (임베딩 벡터 자체는 ChromaDB에 별도 저장)
4) 생성 작업(문서)과 입력 컨셉·사용 모델·근거율·진행 상태·출력 경로를 저장한다.
5) 문서별로 채운 필드 값과 근거 여부(grounded)·신뢰도를 저장한다.
6) 채운 값과 근거가 된 지식 청크의 다대다(M:N) 관계를 저장해, "어떤 값이 어떤 문서에 근거했는지" 추적을 지원한다.
7) KPI 평가 결과(채움 정확도·근거 일치율·시간 단축·외부 호출 수)를 저장한다.

## 2. 객체 정의서

| 객체 명 | 속성 명 |
|---|---|
| users (사용자) | user_id(PK), email, name, created_at |
| forms (빈 양식) | form_id(PK), user_id(FK), filename, page_count, is_form, src_path, created_at |
| form_fields (폼필드) | field_id(PK), form_id(FK), name, label, field_type, page, bbox_x0, bbox_y0, bbox_x1, bbox_y1, required, max_length |
| knowledge_documents (지식 문서) | doc_id(PK), source, title, char_count, indexed_at |
| knowledge_chunks (지식 청크) | chunk_id(PK), doc_id(FK), chunk_index, text, char_count |
| documents (생성 문서) | document_id(PK), form_id(FK), user_id(FK), concept, model, grounded_ratio, status, output_path, created_at |
| filled_fields (채운 필드) | filled_id(PK), document_id(FK), field_name, value, grounded, confidence |
| document_evidence (근거 연결) | filled_id(PK,FK), chunk_id(PK,FK), score |
| eval_results (KPI 평가) | eval_id(PK), dataset, accuracy, time_reduction, grounded_ratio, external_calls, measured_at |

> 컬럼별 타입·제약 상세는 [테이블 명세서](테이블명세서.md), DDL은 [erd.txt](erd.txt) 참조.

## 3. E-R Diagram

![E-R Diagram](diagrams/05_erd.png)

```mermaid
erDiagram
    users ||--o{ forms : "업로드"
    users ||--o{ documents : "요청"
    forms ||--o{ form_fields : "추출"
    forms ||--o{ documents : "생성"
    documents ||--o{ filled_fields : "포함"
    knowledge_documents ||--o{ knowledge_chunks : "청킹"
    filled_fields ||--o{ document_evidence : "근거"
    knowledge_chunks ||--o{ document_evidence : "근거"

    users {
        int user_id PK
        varchar email
        varchar name
    }
    forms {
        varchar form_id PK
        int user_id FK
        varchar filename
        int page_count
    }
    form_fields {
        int field_id PK
        varchar form_id FK
        varchar name
        varchar label
        varchar field_type
    }
    knowledge_documents {
        int doc_id PK
        varchar source
        int char_count
    }
    knowledge_chunks {
        varchar chunk_id PK
        int doc_id FK
        text text
    }
    documents {
        int document_id PK
        varchar form_id FK
        int user_id FK
        text concept
        decimal grounded_ratio
    }
    filled_fields {
        int filled_id PK
        int document_id FK
        varchar field_name
        text value
        bool grounded
    }
    document_evidence {
        int filled_id PK "FK to filled_fields"
        varchar chunk_id PK "FK to knowledge_chunks"
        decimal score
    }
    eval_results {
        int eval_id PK
        varchar dataset
        decimal accuracy
    }
```

| 관계 | 설명 |
|---|---|
| users 1:N forms / documents | 사용자가 여러 양식 업로드·문서 생성 |
| forms 1:N form_fields / documents | 양식의 추출 필드, 양식으로 만든 문서 |
| documents 1:N filled_fields | 문서별 채운 값 |
| knowledge_documents 1:N knowledge_chunks | 문서 → 청크 분할 |
| filled_fields N:M knowledge_chunks (document_evidence) | 채운 값 ↔ 근거 청크 (근거 추적) |
