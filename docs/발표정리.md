# 🎤 뚝딱(TookTak) 발표 정리

> **빈 PDF 양식 + 컨셉 한 줄 입력만으로 완성 문서를 만드는 — 외부 전송 0건, 완전 로컬 AI 문서 어시스턴트**

기본 정보: 팀 **뚝딱** (김세경·채요한·박은선·이권형) / 엑시오 기업과제(PBL) / 멘토 김전일 / 4주

---

## 슬라이드 흐름 (12장)

### 1. 표지
제목 + 한 줄 소개 + 팀명/팀원/기업.

### 2. 배경 — 왜 지금 이게 필요한가
- 생성형 AI 도입 **55.7% → 2026년 85%** 전망, 1위 활용은 **문서 작성**
- 동시에 **보안 우려 1위(81.2%)** · 결과 신뢰도 부족(61.3%)
- **온프레미스 추론 12%(2023) → 55%(2025)** 4.6배 증가
- 💬 *"AI는 쓰고 싶은데 데이터는 못 내보낸다 — 이 모순이 우리 출발점"*

### 3. 문제 정의
- 정형 양식(견적서·신청서·사업계획서) 작성에 반복 수작업
- 상용 AI는 사내 데이터 유출 위험으로 도입 제약
- 기존 도구는 **요약·Q&A 중심**, "빈 양식을 인식해 채우는" 자동화는 미비

### 4. 솔루션 — 처리 흐름
```
빈 양식 업로드 → PDF 파싱 → RAG 검색 → LLM 생성 → 좌표 한글 주입 → 미리보기·다운로드
```
💬 *"전 구간이 사내에서만 동작 — 외부 네트워크 호출 0건"*

### 5. 🎬 데모 (★ 발표의 핵심 — 아래 시연 순서 참고)

### 6. 시스템 아키텍처
- 계층 분리: 프론트(React) → API(FastAPI) → 처리(파싱·RAG·생성·주입) → 데이터(Qdrant)
- 단방향 의존(DAG), **인터페이스 계약(schemas) 먼저 합의 → 4명 병렬 개발**

### 7. 기술 스택 & 모델 선정
| 영역 | 선택 | 이유 |
|---|---|---|
| LLM | **Qwen3 8B** (Apache-2.0) | 상업 이용 자유, 한국어 양호, VRAM ~6GB |
| 임베딩 | BGE-M3 (1024d) | 다국어·한영 혼합 강함 |
| 파싱/주입 | PyMuPDF + pypdf | 좌표 정밀 |
| 벡터DB | Qdrant | 로컬·주니어 친화 |

💬 *"EXAONE이 한국어는 더 좋지만 비상업 라이선스라, 사업화 위해 Qwen3 채택"*

### 8. 차별점 (경쟁 비교)
| | ChatGPT/Copilot | Adobe AI | **뚝딱** |
|---|:---:|:---:|:---:|
| 빈 양식 자동 채움 | ✕ | △ | **✓** |
| 온프레미스 완전 로컬 | ✕ | ✕ | **✓ (유출 0)** |
| 좌표 정밀 주입 | ✕ | △ | **✓** |
| 근거/추론 구분 | ✕ | ✕ | **✓** |

### 9. 📊 KPI 실측 (3종 문서) — *정직하게*
| KPI (합격선) | 견적서·단순 | 견적서·표준 | 사업계획서 |
|---|:---:|:---:|:---:|
| 채움 정확도 (≥80%) | 93% | 100% | 100% |
| 시간 단축 (≥50%) | 98% | 99% | 100% |
| 근거 일치율 (≥90%) | 100% | 88%¹ | 100% |
| 외부 호출 0건 | 0 | 0 | 0 |

¹ *88%는 자동 계산값(부가세·합계) 2개 때문 — 값은 정확, strict 매칭에서만 빠짐. 조작 없이 실측 그대로 보고.*

💬 *"외부 호출 0건은 말이 아니라 소켓을 가로채 실측으로 증명"*

### 10. 엔지니어링 하이라이트 (실제로 푼 문제들)
- **한글 폰트 주입** — 기획서가 경고한 최대 함정. 좌표 오버레이 + 내장 CJK 폰트로 해결
- **다열 라벨 추론** — 옆 칸 라벨 흡수 버그를 "가장 가까운 묶음만"으로 수정
- **긴 값/문단 렌더링** — 자동 글자 축소 + 멀티라인 줄바꿈
- **테스트 38개** — 가짜 LLM/임베더 주입으로 **GPU 없이도** 전부 통과

### 11. 활용 & 로드맵
- **활용**: 정부지원사업 사업계획서, 사내 견적서·보고서, 금융·의료·공공 프라이빗 문서
- **중장기**: 평면 PDF 주입 · 하이브리드 검색+리랭킹 · 멀티모달 · B2B 패키지화

### 12. 마무리
💬 *"기획서 한 장에서 출발해 — 3종 문서를 정확도 93~100%로 채우고, 외부 전송 0건을 실측으로 증명한 동작하는 시스템을 만들었습니다"*

## 🎬 데모 시연 순서 (실시간 추천)

> 사전 준비: `docker compose up` 또는 백엔드+프론트 기동, Ollama 실행 확인

1. **웹 UI 접속** (localhost:5173) — 업로드 화면
2. **견적서 양식 업로드** → 필드 27개 자동 추출 보여주기
3. **컨셉 한 줄 입력** → "엑시오가 ABC상사에 클라우드 서버 구축 견적, 2대 300만원"
4. **생성** → 미리보기에서 **`근거`(초록)/`추론`(주황) 배지** 강조 — *"공급가액 600만, 부가세 60만은 모델이 계산"*
5. **PDF 다운로드** → 완성된 견적서 열기
6. **킬러 한 방**: *"그럼 우리 기획서를 넣으면?"* → **기획서 → 사업계획서 신청서 자동 작성** (근거율 100%)

> 💡 실시간 LLM이 불안하면 미리 만든 스크린샷(`docs/screenshots/`)으로 백업

## ❓ 예상 질문 Q&A

| 질문 | 답변 포인트 |
|---|---|
| **정말 데이터가 안 나가나요?** | 소켓 트래픽을 가로채 외부 도메인 호출 0건을 자동 측정. KPI로 증명 |
| **근거율 88%는 실패 아닌가요?** | 정확도는 100%. 88%는 자동 계산값 2개가 strict 매칭에서 빠진 것 — 환각 아님. 정직하게 보고 |
| **상용 ChatGPT보다 나은 점?** | 빈 양식을 좌표 단위로 채움 + 데이터 외부 유출 0 + 근거 추적 가능 |
| **주니어 4주에 가능했나?** | 인터페이스(schemas) 먼저 합의 → 병렬 개발. 모듈마다 스텁으로 처음부터 E2E 동작 |
| **양식이 바뀌면?** | 표준 견적서(27필드)·사업계획서로 검증. 다열·문단·계산까지 대응. 평면 PDF는 중장기 |
| **GPU 없으면?** | CPU 폴백. 임베딩은 CPU 권장. 테스트는 가짜 LLM으로 GPU 없이 통과 |

## 핵심 숫자 (암기용)

- 문서 3종 채움 정확도 **93~100%**
- 외부 호출 **0건** (소켓 실측)
- 근거 일치율 **88~100%**
- 테스트 **38개** 통과
- LLM **Qwen3 8B** (Apache-2.0), 임베딩 1024차원
- RTX 4060 Ti에서 VRAM ~5.4GB, ~57 tok/s
